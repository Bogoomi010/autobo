# Agent Instructions

## 빌드 규칙

- 항상 작업이 끝난 뒤에는 `npm run build`가 아닌 `npm run tauri build`를 진행합니다.

이 프로젝트에서 작업할 때는 코드 변경 전에 `docs` 경로의 문서를 먼저 확인한다.

## 필수 참조 문서

- `docs/upbit-api-implementation-notes.md`
  - Upbit API 전역 구현 주의사항
  - 인증, JWT, `query_hash`, rate limit, REST/WebSocket 에러 처리
  - Tauri 구조에서 API Key와 API 호출을 다루는 원칙

- `docs/upbit-endpoint-catalog.md`
  - Upbit REST API 엔드포인트별 구현 메모
  - Quotation, Exchange, 주문, 입출금, 서비스 정보 API
  - endpoint별 path, 권한, rate limit, 파라미터, 구현 주의사항

## 작업 원칙

- Upbit API 연동 코드를 작성하거나 수정하기 전에는 위 문서를 확인한다.
- 공식 문서와 로컬 `docs` 내용이 충돌하면 공식 문서를 다시 확인하고 `docs`를 갱신한다.
- 인증이 필요한 API는 React 프론트엔드에서 직접 호출하지 않고 Tauri Rust 백엔드에서 처리한다.
- API Key, JWT, 계정 정보, 주문/입출금 응답의 민감 정보는 로그와 UI에 그대로 노출하지 않는다.
- 주문, 입금, 출금처럼 자산 이동이 발생하는 기능은 dry-run, 테스트 API, 명시적 사용자 확인, 권한 분리를 우선한다.
- 자동매매 전략 코드에서 입금/출금 요청 API를 직접 호출하지 않는다.
- rate limit은 endpoint 단위가 아니라 Upbit 문서의 rate limit group 기준으로 설계한다.
- `query_hash`가 필요한 요청은 실제 요청 파라미터와 해시 입력 문자열의 순서와 구성을 일치시킨다.

## 비트코인 자동투자봇 설계 지침

이 프로젝트는 Upbit API 기반 비트코인 자동투자봇이다.

자동매매 기능을 구현할 때는 `docs/BTC_BOT_AGENT_CONTEXT.md`를 먼저 확인한다.  
해당 문서는 백테스트, 데이터 저장, 전략 구조, 성과 측정, 리스크 관리, 이벤트 기반 트레이딩 엔진 설계 기준을 담고 있다.

### 자동매매 구현 원칙

- 실거래 주문 기능보다 데이터 수집, 백테스트, 모의투자, dry-run을 먼저 구현한다.
- 백테스트와 실거래는 가능한 한 같은 전략 코드 경로를 사용하도록 설계한다.
- 과거 데이터를 한 번에 모두 참조하는 방식으로 전략을 만들지 않는다.
- 전략은 시간순 데이터 흐름 기준으로 동작해야 하며 look-ahead bias를 만들지 않는다.
- 수수료, 스프레드, 슬리피지, 최소 주문금액, 부분체결 가능성을 백테스트에 반영한다.
- 단순 수익률만 보지 말고 BTC 단순보유 대비 초과수익, MDD, Sharpe, Sortino, 거래 횟수, 수수료 총액을 함께 계산한다.
- 초기 전략은 DCA 기준 전략을 먼저 만들고, 이후 이동평균 필터와 변동성 필터를 단계적으로 추가한다.
- 머신러닝/예측 모델은 초기 매수·매도 핵심 로직으로 사용하지 않고 보조 필터로만 검토한다.
- 파라미터 최적화 시 특정 구간 최고 수익률이 아니라 여러 기간에서 안정적인 파라미터 영역을 찾는다.
- 실거래 주문 전에는 반드시 모의투자 또는 dry-run 결과와 로그를 남긴다.
- 자동매매 전략 코드에서는 입금/출금 API를 호출하지 않는다.
- 자동매매 루프에는 수동 중지 스위치, 일일 손실 제한, 최대 낙폭 제한, API 오류 연속 발생 시 중단 조건을 둔다.

### 권장 구현 순서

1. 현재 repo 구조와 기존 Upbit API 구현 확인
2. 데이터 수집 계층 구현 또는 정리
3. OHLCV, 체결, 호가, 주문, 잔고, 오류 로그 저장 구조 설계
4. 백테스트 엔진 구현
5. DCA 기준 전략 구현
6. 수수료, 슬리피지, 최소 주문금액 반영
7. 성과 측정 리포트 구현
8. 모의투자/dry-run 실행 루프 구현
9. 리스크 관리 규칙 구현
10. 실거래 주문은 마지막 단계에서 별도 승인 후 구현
