# 편의 기능 설계 계획: 수동 주문 빠른 입력

## 선정 기능

수동 주문 패널에 빠른 입력 도구를 추가한다. 사용자는 현재가, 시장가 매수 금액, 시장가 매도 수량, 지정가 주문 수량을 버튼으로 채울 수 있다.

## 선정 이유

- 수동 주문은 실사용 빈도가 높고, 가격과 금액과 수량을 반복 계산해 입력해야 한다.
- 기존 자동화에서 구현한 화면 설정 저장과 겹치지 않는다.
- 현재 앱은 시세와 주문 가능정보를 이미 조회하므로 추가 API 없이 입력 편의성을 개선할 수 있다.
- API Key, 주문 실행 상태, 계좌 정보 저장 정책에는 영향을 주지 않는다.

## 설계

1. `get_order_chance` 응답에서 매수 가능 기준통화 잔고와 매도 가능 코인 잔고를 안전하게 읽는다.
2. 수동 주문 패널에 빠른 입력 영역을 추가한다.
3. 지정가 또는 최유리 주문 준비 중에는 현재 티커 가격을 가격 입력칸에 채울 수 있게 한다.
4. 시장가 매수용 고정 금액 버튼은 선택 종목을 현재 종목으로 맞추고 `side=bid`, `ord_type=price`로 전환한 뒤 매수 금액을 채운다.
5. 주문 가능정보가 있으면 25%, 50%, 100% 비율 버튼을 제공한다.
6. 지정가 주문에서는 현재가가 있을 때 금액을 수량으로 환산해 입력한다.
7. 빠른 입력 버튼은 기존 `manualOrder` 상태를 갱신하므로 사용자 설정 저장 로직과 자연스럽게 연동된다.
8. 현재가가 아직 로딩되지 않았거나 유효하지 않으면 현재가 기반 버튼은 비활성화한다.

## 제외 범위

- 실제 주문 전송 로직 변경
- API Key 저장 방식 변경
- Upbit 최소 주문 금액, 호가 단위 보정, 수수료 계산
- 입금/출금 API 호출

## 검증 기준

- `npm run tauri build` 통과
- 빠른 입력 로직이 API Key, 계좌 원문 응답, 로그 저장 범위를 늘리지 않음
- 현재가 버튼이 티커 가격을 문자열로 입력 상태에 반영
- KRW 금액 버튼이 시장가 매수 프리셋과 매수 금액을 함께 반영
- 기존 화면 설정 저장 로직과 충돌하지 않음
