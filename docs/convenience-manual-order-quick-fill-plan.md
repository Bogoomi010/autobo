# 편의 기능 설계 계획: 수동 주문 빠른 입력

## 선정 기능

수동 주문 패널에 현재가 입력 버튼과 자주 쓰는 KRW 시장가 매수 금액 버튼을 추가한다.

## 선정 이유

- 사용자는 주문할 때 현재 선택 종목의 시세를 확인한 뒤 가격 또는 매수 금액을 반복 입력해야 한다.
- 기존 화면 설정 자동 저장 기능과 겹치지 않으며, 실제 주문 전 입력 단계를 줄이는 기능적 편의성이 크다.
- API Key, 주문 실행 상태, 계좌 정보 저장 정책에는 영향을 주지 않는다.

## 동작 설계

1. 지정가 또는 최유리 주문 준비 중에는 현재 티커 가격을 가격 입력칸에 채울 수 있게 한다.
2. 시장가 매수 금액 프리셋 버튼은 선택 종목을 현재 종목으로 맞추고 `side=bid`, `ord_type=price`로 전환한 뒤 매수 금액을 채운다.
3. 빠른 입력 버튼은 기존 `manualOrder` 상태를 갱신하므로 이전 자동화에서 구현한 사용자 설정 저장 로직과 자연스럽게 연동된다.
4. 현재가가 아직 로딩되지 않았거나 유효하지 않으면 현재가 버튼은 비활성화한다.

## 제외 범위

- 잔고 기반 전량 매도 또는 보유 비율 매도 자동 계산은 이번 범위에서 제외한다.
- Upbit 최소 주문 금액, 호가 단위 보정, 수수료 계산은 기존 주문 검증 흐름에 추가하지 않는다.
- API Key와 Secret Key는 계속 저장하지 않는다.

## 검증 기준

- `npm run build` 통과
- 현재가 버튼이 티커 가격을 문자열로 입력 상태에 반영
- KRW 금액 버튼이 시장가 매수 프리셋과 매수 금액을 함께 반영
- 기존 화면 설정 저장 로직과 충돌하지 않음
