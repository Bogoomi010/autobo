# 편의 기능 설계 계획: 수동 주문 빠른 입력

## 선정 기능

수동 주문 패널에 빠른 입력 도구를 추가한다. 사용자는 시장가 매수 금액, 시장가 매도 수량, 지정가 주문 수량을 버튼으로 채울 수 있다.

## 선정 이유

- 수동 주문은 실사용 빈도가 높고, 금액과 수량을 매번 계산해 입력해야 한다.
- 기존 자동화에서 구현한 화면 설정 저장과 겹치지 않는다.
- 현재 앱은 시세와 주문 가능정보를 이미 조회하므로 추가 API 없이 입력 편의성을 개선할 수 있다.

## 설계

1. `get_order_chance` 응답에서 매수 가능 기준통화 잔고와 매도 가능 코인 잔고를 안전하게 읽는다.
2. 수동 주문 패널에 빠른 입력 영역을 추가한다.
3. 시장가 매수용 고정 금액 버튼을 제공한다.
4. 주문 가능정보가 있으면 25%, 50%, 100% 비율 버튼을 제공한다.
5. 지정가 주문에서는 현재가가 있을 때 금액을 수량으로 환산해 입력한다.
6. 주문 가능정보가 없으면 계좌/주문 가능정보 갱신을 안내하고 고정 금액 버튼은 계속 사용할 수 있게 한다.

## 제외 범위

- 실제 주문 전송 로직 변경
- Upbit 최소 주문 금액 또는 호가 단위 검증
- API Key 저장 방식 변경

## 검증 기준

- `npm run build` 통과
- `cargo check` 통과
- 빠른 입력 로직이 API Key, 계좌 원문 응답, 로그 저장 범위를 늘리지 않음
