# 편의 기능 설계 계획: 수동 주문 도우미

## 선정 기능

수동 주문 패널에 주문 도우미를 추가한다. 사용자는 현재가를 가격 입력칸에 바로 채우고, 자주 쓰는 주문 금액으로 수량을 계산하며, 전송 전 주문 조합과 예상 주문 금액을 확인할 수 있다.

## 선정 이유

- 수동 주문은 가격, 수량, 주문 유형 조합을 직접 맞춰야 해서 입력 실수 가능성이 높다.
- 기존 자동화 작업은 화면 설정 자동 저장이므로 주문 입력을 돕는 기능과 범위가 겹치지 않는다.
- API Key나 계좌 정보를 저장하지 않고도 프론트엔드에서 즉시 제공할 수 있는 기능적 편의성이다.

## 구현 범위

- 현재 선택 마켓의 현재가를 수동 주문 가격에 채우는 버튼 추가
- 10,000 / 50,000 / 100,000 KRW 금액 버튼 추가
- 지정가 주문에서는 금액과 가격으로 수량 자동 계산
- 시장가 매수에서는 금액을 매수 금액으로 입력
- 시장가 매도에서는 현재가 기준으로 해당 금액만큼의 수량 계산
- 주문 유형별 필수 입력과 side/ord_type 조합을 전송 전에 확인
- 주문 예상 금액, 현재 마켓, 주문 기준을 요약 표시

## 제외 범위

- 계좌 잔고 기반 최대 매수/매도 계산
- 실제 Upbit 최소 주문 금액과 호가 단위 검증
- 자동 전략 주문 로직 변경
- API Key, 계좌, 주문 가능 정보 저장

## 검증 기준

- `npm run build` 통과
- TypeScript 타입 오류 없음
- 잘못된 수동 주문 조합은 API 호출 전에 로그 경고로 차단
- 기존 설정 저장, 즐겨찾기, 차트, 자동 전략 흐름과 충돌하지 않음
