# 편의 기능 설계 계획: 화면 설정 자동 저장

## 선정 기능

사용자가 반복 입력하는 화면 설정을 브라우저 `localStorage`에 자동 저장하고 다음 실행 때 복원한다.

## 선정 이유

- 사용자는 매번 같은 마켓, 차트 주기, 전략 조건, 주문 입력값을 다시 맞춰야 한다.
- 기존 기능인 마켓 목록, 차트, 자산 창과 겹치지 않고 모든 주요 화면 흐름의 반복 조작을 줄인다.
- API Key는 저장하지 않아 현재 보안 정책을 유지할 수 있다.

## 저장 대상

- 선택 마켓
- 모의 실행 여부
- 마켓 필터, 검색어, 정렬 방식
- 차트 주기
- 자동 전략 입력값
- 수동 주문 입력값

## 제외 대상

- Access Key, Secret Key
- 실행 중 상태
- 잔고, 주문 가능정보, 최근 응답, 로그

## 구현 계획

1. 기본 설정 객체와 저장 키를 `src/App.tsx`에 정의한다.
2. 저장값을 읽을 때 타입과 허용값을 검증하고, 잘못된 값은 기본값으로 대체한다.
3. 주요 설정 상태의 초기값을 저장값에서 가져오도록 변경한다.
4. 설정 상태가 바뀔 때마다 `localStorage`에 저장한다.
5. API 키 패널에 설정 초기화 버튼을 추가해 기본값으로 되돌릴 수 있게 한다.
6. `src/App.css`에 작은 보조 액션 스타일을 추가한다.

## 검증 기준

- `npm run build` 통과
- TypeScript 타입 오류 없음
- 저장/복원 로직이 API Key를 포함하지 않음
