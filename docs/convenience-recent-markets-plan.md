# 편의 기능 설계 계획: 최근 선택 종목 바로가기

## 선정 기능

사용자가 최근에 선택하거나 입력한 마켓 코드를 자동으로 기록하고, 상단 시세 영역 아래에서 버튼으로 다시 선택할 수 있게 한다.

## 선정 이유

- 자동매매 앱에서는 여러 종목을 번갈아 차트, 현재가, 주문 입력에 적용하는 작업이 잦다.
- 즐겨찾기는 사용자가 직접 관리해야 하지만, 최근 선택 목록은 별도 관리 없이 반복 이동을 줄인다.
- API Key, 잔고, 주문 응답 같은 민감 정보와 무관하며 기존 화면 설정 자동 저장 기능과 저장 목적이 다르다.

## 저장 범위

- 저장 대상: 유효한 마켓 코드 목록
- 저장 위치: 브라우저 `localStorage`
- 저장 개수: 최근 6개
- 제외 대상: API Key, 계좌 정보, 주문 응답, 로그, 자동 감시 실행 상태

## 구현 계획

1. `src/App.tsx`에 최근 마켓 저장 키, 최대 개수, 로드 함수를 추가한다.
2. 선택 마켓이 유효한 코드로 바뀔 때 최근 목록 맨 앞에 추가하고 중복을 제거한다.
3. 최근 목록을 `localStorage`에 저장한다.
4. 상단 마켓 영역 아래에 최근 종목 버튼과 비우기 버튼을 추가한다.
5. `src/App.css`에 최근 종목 영역과 버튼 스타일을 추가한다.

## 검증 기준

- `npm run build` 통과
- 최근 종목 로드/저장 로직이 잘못된 저장값을 무시한다.
- 최근 종목 버튼 클릭 시 선택 마켓이 바뀌고 기존 차트/시세 갱신 흐름을 그대로 사용한다.
