# 🏢 코인 오피스 — GDD (v2.0 · 실거래)

일본 도트 감성 타이쿤 게임. 탑다운 사무실에서 캐릭터를 조작해
**실제 업비트 계좌로 코인을 매매**한다.

- **Tauri 데스크톱 = 실거래 모드**: 금고 잔액 = 실제 계좌 KRW, 투자 = 실제 시장가 매수,
  ±3% 자동매도 = 실제 시장가 매도. ⚠ 게임 조작이 곧 실제 주문이다.
- **브라우저 dev = 모의 모드**: 가상 잔고 100만원 + 실시세. UI/게임플레이 개발용.

기술 스택: TypeScript + Vite + Phaser 3 + Tauri 2 (Rust 업비트 클라이언트는 기존 autobo 것 재사용)

## 핵심 루프

1. **금고방**에서 잔액을 **만원 단위**로 꺼낸다 (머리 위 셀렉터 UI — 출금은 연출, 실 돈 이동 없음)
2. 돈뭉치를 **양손으로 들고** (스프라이트 변경 + 머리 위 돈뭉치) 투자방으로 이동
3. 투자방 **코인 단말기**와 상호작용 → **코인 시세판 모달**(도트 감성 오버레이)
4. 코인 선택 → 확인 화면(⚠ 실거래 경고) → 들고 있는 금액 전액 **시장가 매수**
   (체결 확인까지 수 초 — 체결 수량·평균가·수수료는 주문 조회로 확정)
5. 시세 폴링(3초, `/v1/ticker/all` 1회) — 평균 체결가 대비 **+3% 익절 / -3% 손절 시 자동 시장가 매도**
6. 정산기가 **실 체결액(수수료 차감)만큼 돈뭉치를 배출** → 주워서(Space) 금고방에 재입금(연출)

## 실거래 안전 원칙 (AGENTS.md·upbit-api-implementation-notes.md 준수)

- 이 게임은 **자신이 매수한 포지션 물량만** 자동 매도한다. 계좌에 원래 있던 코인은 불간섭.
- 앱 시작 시 저장된 포지션을 실제 코인 잔고와 대조 — 외부에서 팔렸으면 자동 정리.
- 주문 생성 응답만 믿지 않고 `GET /v1/order`(get_order 커맨드)로 최종 체결을 확인.
- 시장가 매수(ord_type=price)는 잔액 반환으로 state가 `cancel`로 끝나는 것이 정상 — done/cancel 모두 종결로 처리.
- `status: "selling"` 가드로 매도 주문 중복 전송 방지. 실패 시 open 복구 후 다음 폴링에서 재시도.
- 매수 진행 중(ORDER_BUSY)에는 모달 닫기/추가 주문 잠금.
- API Key는 실행 파일 옆 `upbitkey` 파일 → Rust 세션 메모리에만 보관 (기존 autobo 방식).
- 입출금 API는 사용하지 않는다.

## 돈 표시 모델 (실거래)

잔고의 원본은 업비트 계좌다. 게임 내 이동은 연출이며 다음 등식으로 표시한다:

```
금고 표시 잔액 = 계좌 KRW − 들고 있는 돈 − 바닥 돈뭉치 합
총 자산       = 금고 + 들고 있는 돈 + 포지션 원금 + 바닥 돈뭉치
```

실 돈 이동은 매수(계좌 KRW ↓)·매도(계좌 KRW ↑)에서만 발생. 매도 직후 KRW는 이미 계좌에
있지만, 돈뭉치를 주워 금고에 넣기 전까지는 금고 표시에서 제외해 게임 루프를 유지한다.

## 업비트 API (전부 기존 Rust 커맨드 경유 — 프론트 직접 호출 금지)

| 용도 | 커맨드 | 비고 |
| --- | --- | --- |
| 마켓 목록 | `get_markets(is_details)` | KRW-, 유의 종목 제외 |
| 전 마켓 시세 | `get_quote_tickers("KRW")` | 3초 폴링, 실패 시 지수 백오프 |
| 계좌 연동 | `connect_upbitkey_account` | upbitkey 파일 → 세션 |
| 잔고 | `get_session_accounts` | 주문 후 갱신 |
| 주문 | `place_order(order, dry_run)` | 시장가 bid(price)/ask(volume), dry_run=false |
| 체결 확인 | `get_order(uuid)` | **신규 추가** — trades/paid_fee 포함 |

브라우저 모의 모드는 Quotation만 vite 프록시(`/upbit-api`)로 호출한다.
