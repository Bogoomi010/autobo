# 편의 기능 구현 결과: 빠른 종목 바로가기

## 구현 내용

- `src/App.tsx`에 최근 선택 종목 저장/복원 로직을 추가했다.
- 최근 종목은 `autobo.recentMarkets.v1` 키로 `localStorage`에 저장하며 최대 8개만 유지한다.
- 선택된 마켓이 실제 마켓 목록에 존재할 때만 최근 종목에 기록하도록 했다.
- 최근 종목과 즐겨찾기 종목을 합쳐 상단 시세 영역 아래에 빠른 바로가기 UI를 표시한다.
- 바로가기는 최근 종목을 우선 배치하고 최대 12개까지만 표시해 상단 영역이 과도하게 길어지지 않게 했다.
- 각 바로가기는 종목명, 마켓 코드, 등락률, 현재가를 표시하고 클릭 시 해당 종목으로 전환한다.
- 즐겨찾기 종목은 별 아이콘으로 구분하고, 최근 종목 초기화 버튼을 추가했다.
- `src/App.css`에 빠른 바로가기 레이아웃, 선택 상태, 말줄임, 반응형 그리드 스타일을 추가했다.

## 보안 및 저장 범위

- Access Key, Secret Key, 주문 요청 본문은 저장하지 않는다.
- 저장되는 값은 마켓 코드 문자열 목록뿐이다.
- 기존 화면 설정 저장 키와 분리된 별도 키를 사용한다.

## 검증 결과

- `npm run build -- --configLoader runner`: 통과
- `cargo check` in `src-tauri`: 통과
- 별도 테스트 스크립트는 `package.json`에 정의되어 있지 않아 실행하지 않았다.

## 참고

- 현재 샌드박스에서는 `npm`이 사용자 전역 prefix와 캐시 로그 경로 권한 문제로 `npm install`을 정상 완료하지 못했다.
- 빌드는 원본 워크스페이스의 설치된 `node_modules`를 참조하고, Vite config bundling의 샌드박스 경로 접근 문제를 피하기 위해 `--configLoader runner` 옵션을 전달해 검증했다.
