# 편의 기능 설계 계획: 수동 주문 입력 보조

## 선정 기능

수동 주문 패널에서 선택 종목의 현재가와 기준 통화를 활용해 주문 입력을 빠르게 채우고 예상 주문 규모를 확인할 수 있게 한다.

## 선정 이유

- 기존 자동화 작업은 화면 설정 자동 저장이므로, 이번 기능은 주문 입력 단계의 반복 계산을 줄이는 별도 편의 기능이다.
- 사용자는 지정가 주문 가격, 시장가 매수 금액, 매도 예상 금액을 매번 현재가 기준으로 계산해야 한다.
- API Key나 주문 실행 권한을 바꾸지 않고도 수동 주문 전 확인 편의성을 높일 수 있다.

## 구현 계획

1. 선택 종목에서 기준 통화와 거래 자산을 추출하는 보조 값을 추가한다.
2. 현재가가 있는 경우 지정가 가격 입력에 현재가를 채우는 액션을 제공한다.
3. 시장가 매수에 자주 쓰는 금액 버튼을 제공하고, 선택 종목의 기준 통화에 맞춰 입력한다.
4. 주문 유형, 가격, 수량, 현재가를 기준으로 예상 매수 수량 또는 예상 주문 금액을 계산해 표시한다.
5. 기존 수동 주문 전송 로직은 변경하지 않고 입력 보조 UI만 추가한다.

## 검증 기준

- `npm run build` 통과
- 기존 주문 요청 payload 변환 로직이 유지됨
- 예상값 계산이 값이 없거나 0 이하인 입력에서 안전하게 대기 상태를 표시함
