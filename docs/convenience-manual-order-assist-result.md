# 편의 기능 구현 결과: 수동 주문 입력 보조

## 구현 내용

- 수동 주문 패널에 선택 종목의 기준 통화와 거래 자산, 현재가 표시를 추가했다.
- 현재가가 있는 경우 지정가 가격 입력에 현재가를 바로 채울 수 있는 버튼을 추가했다.
- 기준 통화별 빠른 시장가 매수 금액 버튼을 추가했다.
  - KRW: 1만, 5만, 10만
  - BTC: 0.0001, 0.001, 0.01
  - USDT: 10, 50, 100
- 주문 유형과 입력값에 따라 예상 매수 수량, 예상 매도 금액, 예상 주문 금액을 표시하도록 했다.
- 주문 전송 로직과 API payload 변환은 변경하지 않았다.

## 검증 결과

- `tsc`: 통과
- `vite build --configLoader runner`: 통과

## 비고

- 기본 `npm` shim이 잘못된 사용자 npm 경로를 참조해 `npm run build`는 실행 단계에서 실패했다.
- 현재 작업트리의 불완전한 `node_modules`는 원본 작업공간의 의존성 디렉터리 junction으로 대체해 검증했다.
- Vite 기본 config bundling은 junction 환경에서 상위 디렉터리 접근 오류가 발생해, Vite의 runner config loader로 빌드했다.
- 커밋은 생성했지만 GitHub 443 연결 실패로 원격 푸시와 PR 생성은 진행하지 못했다.
