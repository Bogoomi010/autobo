# 편의 기능 설계 계획: 수동 주문 금액 도우미

## 선정 기능

수동 주문 영역에 현재 선택 종목의 현재가를 기준으로 주문 입력값을 빠르게 채우고 예상 주문 규모를 확인하는 도우미를 추가한다.

## 선정 이유

- 사용자는 수동 주문 시 현재가를 별도로 확인한 뒤 가격, 금액, 수량을 반복 입력해야 한다.
- 기존 자동화 기능은 화면 설정 저장이므로, 이번 기능은 주문 입력 자체의 반복 작업을 줄이는 기능으로 겹치지 않는다.
- 실제 주문 전송 로직은 바꾸지 않고 입력 보조와 계산 표시만 추가하므로 기능적 편의성을 높이면서 위험 범위를 제한할 수 있다.

## 구현 범위

1. 현재 선택 종목의 `ticker.trade_price`를 기준으로 수동 주문 도우미 값을 계산한다.
2. 지정가 주문에는 현재가를 가격 입력칸에 채우는 버튼을 제공한다.
3. 시장가 매수에는 1만원, 5만원, 10만원 빠른 입력 버튼을 제공한다.
4. 가격과 수량이 있는 주문은 예상 주문 금액을 표시한다.
5. 시장가 매수 금액과 현재가가 있는 경우 예상 매수 수량을 표시한다.
6. 시장가 매도 수량과 현재가가 있는 경우 예상 매도 금액을 표시한다.
7. 보조 UI 스타일을 `src/App.css`에 추가한다.

## 제외 범위

- 실제 주문 API 파라미터 구조 변경
- 거래 수수료, 호가 단위, 최소 주문 금액 검증
- 자동 전략 로직 변경

## 검증 계획

- `npm run build`로 TypeScript 및 Vite 빌드 통과 여부를 확인한다.
- 빌드가 통과하면 변경 내용을 커밋한다.
