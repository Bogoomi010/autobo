# 편의 기능 설계 계획: 빠른 종목 바로가기

## 선정 기능

최근 선택한 종목과 즐겨찾기 종목을 상단 시세 영역에서 바로 전환할 수 있는 "빠른 종목 바로가기"를 추가한다.

## 선정 이유

- 사용자는 종목 검색, 즐겨찾기 필터, 목록 스크롤을 반복하지 않고 자주 보는 종목으로 이동할 수 있어야 한다.
- 기존 자동화에서 구현된 화면 설정 자동 저장과 겹치지 않고, 저장된 선호값 위에 실제 탐색 흐름을 줄이는 기능이다.
- API Key나 주문 정보와 무관한 클라이언트 편의 기능이라 보안 범위를 넓히지 않는다.

## 동작 설계

- 선택된 정상 마켓 코드를 최근 종목 목록에 자동 저장한다.
- 최근 종목은 `localStorage`의 별도 키에 최대 8개까지 보관한다.
- 상단 마켓 영역 아래에 빠른 바로가기 줄을 표시한다.
- 바로가기에는 최근 종목을 우선 표시하고, 최근 목록에 없는 즐겨찾기 종목을 이어서 표시한다.
- 바로가기 줄은 화면 과밀을 피하기 위해 최대 12개까지만 표시한다.
- 각 바로가기는 종목 코드, 한글명, 현재가 또는 등락률을 간단히 보여준다.
- 선택 중인 종목은 시각적으로 강조한다.
- 최근 종목 전체 초기화 버튼을 제공한다.

## 제외 범위

- Access Key, Secret Key 저장
- 주문 실행 히스토리 저장
- 서버 또는 Rust 백엔드 변경
- 즐겨찾기 저장 방식 변경

## 구현 계획

1. `src/App.tsx`에 최근 종목 저장/복원 유틸리티와 상태를 추가한다.
2. 선택 마켓이 유효한 코드일 때 최근 종목 목록을 갱신한다.
3. 최근 종목과 즐겨찾기 종목을 합친 바로가기 표시 목록을 만든다.
4. 상단 마켓 영역 아래에 바로가기 UI를 추가한다.
5. `src/App.css`에 빠른 바로가기 스타일과 반응형 레이아웃을 추가한다.

## 검증 기준

- `npm run build` 통과
- TypeScript 오류 없음
- 최근 종목 저장 키가 API Key나 주문 민감 정보를 포함하지 않음
- 즐겨찾기/최근 종목이 없는 초기 상태에서도 UI가 깨지지 않음
