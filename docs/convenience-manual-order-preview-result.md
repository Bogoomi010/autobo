# 편의 기능 구현 결과: 수동 주문 사전 점검

## 구현 내용

- `src/App.tsx`에 수동 주문 검증 유틸리티를 추가했다.
- 주문 방식별로 필수 입력값과 허용되는 `side` 조합을 확인한다.
- 수동 주문 패널에 전송 가능 여부, 주문 요약, 예상 기준 금액, 오류/경고 문구를 표시한다.
- 검증 실패 시 주문 전송 버튼을 비활성화하고, 핸들러에서도 전송 전에 동일 검증을 수행한다.
- 시장가 매수/매도처럼 입력값 일부가 전송되지 않는 주문은 경고로 안내한다.
- 전송 직전 주문 타입별로 사용하지 않는 필드를 `null`로 정규화해 안내와 실제 요청이 어긋나지 않게 했다.
- `src/App.css`에 주문 사전 점검 영역의 ready/blocked 상태 스타일을 추가했다.
- 샌드박스에서 Vite 설정 파일 번들링이 상위 디렉터리 접근 오류를 내지 않도록 `package.json`의 build 스크립트에 `--configLoader runner`를 적용했다.

## 검증 결과

- `npm run build`: 통과
- `cargo check` in `src-tauri`: 통과

## 비고

- 이 기능은 거래소의 최소 주문 금액, 호가 단위, 수수료까지 계산하지 않는다.
- 계좌 잔고 기반 검증은 주문 가능정보 구조를 별도로 타입화한 뒤 추가하는 후속 기능으로 남겼다.
