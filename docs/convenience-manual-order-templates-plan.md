# 편의 기능 설계 계획: 수동 주문 템플릿

## 선정 기능

사용자가 자주 쓰는 수동 주문 조합을 이름 붙여 저장하고, 이후 한 번의 클릭으로 수동 주문 폼에 불러올 수 있는 템플릿 기능을 추가한다.

## 선정 이유

- 기존 화면 설정 자동 저장은 마지막 입력값 복원에는 유용하지만, 여러 주문 패턴을 오가며 재사용하는 흐름은 지원하지 않는다.
- 수동 주문은 종목, 매수/매도, 주문 유형, 가격/금액, 수량, 체결 조건을 반복 입력해야 하므로 기능적 편의성 개선 효과가 크다.
- API Key나 주문 실행 상태를 저장하지 않고 브라우저 로컬 저장소에만 템플릿을 보관해 기존 보안 범위를 유지할 수 있다.

## 저장 범위

- 템플릿 이름
- market
- side
- ord_type
- price
- volume
- time_in_force

## 제외 범위

- identifier는 재사용 시 중복 주문 식별자가 될 수 있어 템플릿 저장 대상에서 제외한다.
- Access Key, Secret Key, 계좌 응답, 주문 응답, 로그는 저장하지 않는다.

## 구현 계획

1. `src/App.tsx`에 수동 주문 템플릿 타입, 기본 로딩/저장/정규화 함수를 추가한다.
2. 템플릿 목록은 `localStorage`의 `autobo.manualOrderTemplates.v1` 키에 최대 5개까지 저장한다.
3. 같은 이름으로 저장하면 기존 템플릿을 덮어쓰고, 새 이름은 최신 항목이 위에 오도록 추가한다.
4. 수동 주문 패널에 템플릿 이름 입력, 저장 버튼, 템플릿 목록, 불러오기/삭제 버튼을 추가한다.
5. 템플릿을 불러올 때는 저장된 주문값을 수동 주문 폼에 반영하되 `identifier`는 현재 입력값을 유지한다.
6. `src/App.css`에 템플릿 목록과 버튼 스타일을 추가한다.

## 검증 기준

- `npm run build` 통과
- 가능한 경우 Tauri Rust 코드 `cargo check` 통과
- 저장/불러오기/삭제 UI가 작은 화면에서도 기존 수동 주문 패널 안에서 흐름을 깨지 않음
