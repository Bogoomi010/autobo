# 편의 기능 설계 계획: 수동 주문 사전 점검

## 선정 기능

수동 주문 패널에 주문 사전 점검 영역을 추가한다. 사용자가 `side`, `ord_type`, 가격, 수량을 조합할 때 전송 전에 필수값 누락과 잘못된 주문 조합을 확인할 수 있게 한다.

## 선정 이유

- 수동 주문은 실제 자금과 직접 연결될 수 있어 입력 실수 방지가 가장 큰 편의 기능이다.
- 이전 자동화에서 구현한 화면 설정 저장, 종목 필터/즐겨찾기와 겹치지 않는다.
- 백엔드 API 계약을 바꾸지 않고 프론트엔드에서 전송 전 검증과 안내를 제공할 수 있다.

## 설계 범위

- 주문 방식별 필수 입력값 검증
  - `limit`: 가격과 수량 필요
  - `price`: 매수 주문만 허용하고 매수금액 필요
  - `market`: 매도 주문만 허용하고 수량 필요
  - `best`: 매수는 가격, 매도는 수량 필요
- 현재 선택 종목과 최신 시세를 기준으로 주문 미리보기 문구 제공
- 검증 실패 시 주문 전송 버튼 비활성화
- 핸들러에서도 동일 검증을 수행해 키보드/상태 경합 상황을 방어
- 시장가 주문에서 무시되는 입력값은 경고 문구로 안내

## 제외 범위

- 거래소의 최소 주문 금액, 호가 단위, 수수료까지 계산하는 정밀 검증
- 계좌 잔고 기반 주문 가능 금액 검증
- 백엔드 주문 API 변경

## 검증 기준

- `npm run build` 통과
- 잘못된 주문 조합이 전송 함수로 넘어가지 않음
- 기존 화면 설정 저장 기능이 새 검증 상태를 별도로 저장하지 않음
